## Project Overview

Static website for the North Carolina Baroque Orchestra (NCBO), built with Jekyll. Data (events, donors, board members) is sourced from Airtable and pulled into YAML files via Python scripts, then rendered by Jekyll/Liquid templates.

## Common Commands

```bash
# Install Ruby dependencies
bundle install

# Local dev server with live reload at http://127.0.0.1:4000
bundle exec jekyll serve --livereload

# Production build (run to verify clean build before committing)
bundle exec jekyll build

# Refresh data from Airtable (requires AIRTABLE_BASE_ID and AIRTABLE_API_KEY env vars)
python refresh_event_data.py   # → _data/events.yml + _data/past_events.yml
python refresh_donors.py       # → _data/donors.yml
python refresh_board.py        # → _data/board.yml
```

There is no automated test suite — use `bundle exec jekyll build` and manual review in the dev server.

## Architecture

**Data flow:** Airtable → Python scripts → `_data/*.yml` → Jekyll/Liquid templates → static HTML

**Key directories:**
- `_layouts/` — Page templates; `home.html` is the primary layout
- `_includes/` — Reusable snippets (events, donors, board, footer, notices)
- `_data/` — YAML data files generated by Python scripts (do not hand-edit)
- `assets/` — CSS/JS bundles; `img/` for images
- `.github/workflows/` — Three automated workflows: fetch Airtable data, deploy to staging, deploy to production

**Deployment pipeline:**
1. GitHub Actions fetches Airtable data → commits YAML to `main`
2. Staging deployment builds and pushes to the `staging` branch (serves `test.ncbaroqueorchestra.org`)
3. Production deployment is manually triggered and pushes to a separate `ncboweb-production` repo (`www.ncbaroqueorchestra.org`)

## Coding Conventions

- 2-space indentation for HTML, Liquid, and YAML
- Front matter must include `layout` and `title`; use lowercase hyphenated keys (e.g., `event-date`)
- Markdown filenames: lowercase with hyphens (`ncbo-history.md`)
- Commit messages: short descriptive summaries under 72 characters (e.g., `Update NCBO events`, `Enhance deployment workflow`)
- Include before/after screenshots in PRs for visible UI changes
